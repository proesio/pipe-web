Eliminar

PIPE tiene dos formas de eliminar datos en las tablas de la base de datos.

Forma #1 de eliminación de datos por medio del modelo.

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	include "Usuario.php"; // Incluimos el archivo Usuario.php que contiene la clase Usuario.
		// El método encontrar() recibe como parámetro el id de la llave primaria.
		$usuario = Usuario::encontrar(1);
		$usuario->eliminar(); // Este método devuelve el número de registros eliminados.

Forma #2 de eliminación de datos por medio del modelo.

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	include "Usuario.php"; // Incluimos el archivo Usuario.php que contiene la clase Usuario.
		$usuario = Usuario::donde('id=? o id=?',[1,2])->eliminar();

Forma #1 de eliminación de datos por medio del constructor de consultas PIPE.

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	use PROES\PIPE;
		// Si nuestra llave primaria no sigue la convención del ORM PIPE podemos especificarla de la siguiente manera.
		$usuario = PIPE::tabla('usuarios')->encontrar(1,'codigo');
		$usuario->eliminar(); // Este método devuelve el número de registros eliminados.

Forma #2 de eliminación de datos por medio del constructor de consultas PIPE.

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	use PROES\PIPE;
		$usuario = PIPE::tabla('usuarios')->donde('id=? o id=?',[1,2])->eliminar();

Eliminación de datos por medio del método consulta().

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	use PROES\PIPE;
		// Este método devuelve el número de registros eliminados.
		$usuario = PIPE::consulta("eliminar de usuarios donde id=1");

Eliminación de datos por medio del método consultaNativa().

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	use PROES\PIPE;
		// Este método devuelve el número de registros eliminados.
		$usuario = PIPE::consultaNativa("delete from usuarios where id=1");

Si utilizas el Modelo para interactuar con los datos de la base de datos, tendras la ventaja de poder omitir las convenciones que tiene el ORM PIPE y adaptarse a la estructura que tenga tu tabla en la base de datos, ya que el Constructor de Consultas se adapta por defecto a las convenciones.
