Insertar

PIPE tiene dos formas de insertar datos en las tablas de la base de datos.

En la forma 1 debemos indicar todos los campos de nuestra tabla por medio de atributos en el objeto a excepción de los campos creado_en y actualizado_en (en caso que estén presentes) debido a que el ORM PIPE se encarga de llenarlos automáticamente.

En la forma 2 podemos decidir qué campos tendrán información que será insertada en la tabla.

Forma #1 de inserción de datos por medio del modelo.

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	include "Usuario.php"; // Incluimos el archivo Usuario.php que contiene la clase Usuario.
		$usuario = Usuario::instanciar();
		$usuario->id = 'null'; // Si nuestra llave primaria es auto incrementable le asignamos una cadena con el valor por defecto en nuestra base de datos.
		$usuario->nombres = 'John';
		$usuario->apellidos = 'Doe';
		$usuario->correo = 'johndoe@correo.com';
		$usuario->insertar(); // Este método devuelve 1 cuando la inserción se ha realizado correctamente.

Forma #2 de inserción de datos por medio del modelo.

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	include "Usuario.php"; // Incluimos el archivo Usuario.php que contiene la clase Usuario.
		// Si nuestra llave primaria es auto incrementable le asignamos una cadena con el valor por defecto en nuestra base de datos.
		$usuario = Usuario::instanciar()->insertar(['id'=>'null','nombres'=>'John','apellidos'=>'Doe','correo'=>'johndoe@correo.com']);

Forma #1 de inserción de datos por medio del constructor de consultas PIPE.

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	use PROES\PIPE;
		$usuario = PIPE::tabla('usuarios');
		$usuario->id = 'null'; // Si nuestra llave primaria es auto incrementable le asignamos una cadena con el valor por defecto en nuestra base de datos.
		$usuario->nombres = 'John';
		$usuario->apellidos = 'Doe';
		$usuario->correo = 'johndoe@correo.com';
		$usuario->insertar(); // Este método devuelve 1 cuando la inserción se ha realizado correctamente.

Forma #2 de inserción de datos por medio del constructor de consultas PIPE.

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	use PROES\PIPE;
		// Si nuestra llave primaria es auto incrementable le asignamos una cadena con el valor por defecto en nuestra base de datos.
		$usuario = PIPE::tabla('usuarios')->insertar(['id'=>'null','nombres'=>'John','apellidos'=>'Doe','correo'=>'johndoe@correo.com']);

Inserción de datos por medio del método consulta().

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	use PROES\PIPE;
		// Este método devuelve 1 cuando la inserción se ha realizado correctamente.
		$usuario = PIPE::consulta("insertar dentro usuarios (id,nombres,apellidos,correo) valores (null,'John','Doe','johndoe@correo.com')");
		// El método consulta() permite enviar los datos de forma segura.
		$usuario = PIPE::consulta('insertar dentro usuarios (id,nombres,apellidos,correo) valores (null,?,?,?)',['John','Doe','johndoe@correo.com']);

Inserción de datos por medio del método consultNativa().

<?php
	require_once "PROES/PIPE_CONEXION_BD.php";
	use PROES\PIPE;
		// Este método devuelve 1 cuando la inserción se ha realizado correctamente.
		$usuario = PIPE::consultaNativa("insert into usuarios (id,nombres,apellidos,correo) values (null,'John','Doe','johndoe@correo.com')");
		// El método consultaNativa() permite enviar los datos de forma segura.
		$usuario = PIPE::consultaNativa('insert into usuarios (id,nombres,apellidos,correo) values (null,?,?,?)',['John','Doe','johndoe@correo.com']);

Si utilizas el Modelo para interactuar con los datos de la base de datos, tendrás la ventaja de poder omitir las convenciones que tiene el ORM PIPE y adaptarse a la estructura que tenga tu tabla en la base de datos, ya que el Constructor de Consultas se adapta por defecto a las convenciones.
